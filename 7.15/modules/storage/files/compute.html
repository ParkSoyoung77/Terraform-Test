02 · Compute Layer

컴퓨팅 (EC2)

네트워크가 만든 경로 위에서 실제 요청을 처리하는 계층입니다. 트래픽을 여러 서버로 나누고, 부하에 따라 서버 수를 자동으로 조절합니다.

LAYER 02 / 03

2.1 EC2와 단일 장애점 문제

EC2 인스턴스 한 대로 서비스를 운영하면 그 인스턴스가 멈추는 순간 서비스 전체가 중단됩니다. 이 문제를 풀기 위해 로드밸런서로 트래픽을 분산하고, Auto Scaling으로 인스턴스 수를 유동적으로 유지하는 구조를 씁니다.

ALB (Application Load Balancer)

들어오는 요청을 여러 대상 그룹의 인스턴스로 분산합니다. 헬스 체크에 실패한 인스턴스로는 트래픽을 보내지 않습니다.

Target Group

ALB가 트래픽을 보낼 인스턴스 목록입니다. 헬스 체크 경로와 주기를 여기서 설정합니다.

2.2 Auto Scaling: 필요한 만큼만 운영

Launch Template에 인스턴스 사양(AMI, 타입, user_data 등)을 정의해두면, Auto Scaling Group이 이 템플릿을 기반으로 인스턴스를 자동으로 늘리거나 줄입니다. 트래픽이 몰릴 때 서버를 늘리고, 한산할 때 줄여 비용을 관리합니다.

TRIGGER CPU 사용률 등 지표가 임계값을 넘으면 스케일링 정책이 동작합니다.
LAUNCH Launch Template 기준으로 새 인스턴스가 생성됩니다.
REGISTER 새 인스턴스가 Target Group에 자동 등록되어 트래픽을 받기 시작합니다.

2.3 무중단 배포: 롤링 방식

새 버전을 배포할 때 전체 인스턴스를 한 번에 교체하면 그 사이 서비스가 끊깁니다. Instance Refresh를 사용하면 일정 비율씩 순차적으로 새 인스턴스로 교체하면서, 항상 최소 가용 용량을 유지합니다.

방식다운타임특징

2.4 실무에서 자주 확인하는 지점
