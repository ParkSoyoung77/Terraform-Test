# CloudFront

1. 1개의 VPC
2. 2개의 가용영역에 각각 1개씩의 퍼블릭서브넷, 프라이빗서브넷, 데이터베이스 전용 서브넷을 생성합니다.
- 총 6개의 서브넷
- 라우팅테이블 총 4개(Default Routing Table 포함)
- 퍼블릭서브넷의 경우 외부와 송수신이 가능하며 외부에서 들어오는 모든 트래픽을 허용합니다.
- 라우팅 테이블은 퍼블릭용 1개, 프라이빗용 1개, 데이터베이스용 1개로 구성합니다.
- 인터넷게이트웨이와 NAT Gateway를 사용합니다.
- 데이터베이스용 서브넷의 경우 외부(인터넷)와의 통신은 완전히 격리 합니다.
3. index.html로 구성된 nginx 프라이빗 인스턴스를 생성하고 최소 1대, 유지 2대, 최대 3대를 동일하게 구성(오토스케일 그룹)
4. 이 오토스케일그룹을 통해 생성된 프라이빗 인스턴스(nginx)에 부하분산 처리를 할 수 있는 단일 진입점을 만듭니다(ALB)
5. 3개의 버킷을 만들고 각각 서로다른 정적웹사이트를 구성합니다.
6. 이 정적 웹사이트를 클라우드프론트로 연결하여 클라우드 프론트의 앤드포인트로 구동될 수 있도록 합니다.
7. MySQL을 인스턴스(1대) 형태로 생성합니다.
- 보안 암호를 이용합니다.
- 사용자가 정의한 서브넷 그룹을 사용합니다.
- 데이터베이스 전용 서브넷에 생성합니다.
- MySQL에 studydb라는 이름의 데이터베이스를 생성합니다.
8. Lambda 함수를 이용하여 데이터베이스에 연결합니다.
- 연결이 성공하면 Success, 실패하면 Failure을 출력합니다.
- 이때 API Gateway를 이용하여 브라우저에 Success / Failure를 보여줍니다.
